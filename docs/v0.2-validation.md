# v0.2 Validation Notes

## Local Validation Commands

```bash
python -m compileall orchestrator
bash -n install.sh uninstall.sh scripts/*.sh
git diff --check
cd orchestrator
python -m pytest
```

## Ubuntu Manual Validation Required Before Merge

Run on Ubuntu Server 24.04 before merging:

```text
PASS: install.sh completes
PASS: scripts/healthcheck.sh exits 0
PASS: GET /health returns ok
PASS: GET /llm/status returns safe JSON
PASS: POST /projects creates project filesystem and DB row
PASS: POST /projects/{project_id}/run completes workflow
PASS: task packets are written to handoffs/
PASS: agent result files are written to handoffs/
PASS: final/final-report.md is created
PASS: GET /projects/{project_id} returns DB-backed metadata and task summary
PASS: no local Ollama is required
```

## Known Limitations

- Formal LangGraph PostgreSQL checkpoint saver integration is deferred to v0.3.
- v0.2 persists workflow state snapshots to PostgreSQL `events`.
- Specialist output is deterministic when Ollama is disabled or unreachable.
- Runtime inspector does not execute shell commands or modify services.

## Recommended v0.3 Issues

- Add a formal LangGraph PostgreSQL checkpoint saver after dependency review.
- Add PostgreSQL-backed integration tests in CI.
- Add a narrow approval workflow for candidate harness and skill promotion.

For v0.3 LLM-backed specialist validation, see `docs/v0.3-validation.md`.
