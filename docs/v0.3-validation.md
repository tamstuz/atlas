## Agent Run Audit Query

`agent_runs` links to projects through `tasks.task_id`. Use this query to validate LLM/fallback metadata for a project:

```sql
select
  ar.agent_role,
  ar.status,
  ar.input->>'model' as input_model,
  ar.input->>'provider' as input_provider,
  ar.output->>'model' as output_model,
  ar.output->>'provider' as output_provider,
  ar.output->>'fallback_used' as fallback_used,
  ar.output->>'error' as error
from agent_runs ar
join tasks t on t.id = ar.task_id
where t.project_id='<PROJECT_ID>'
order by ar.created_at;
```

## Known Limitations

- Ollama remains external and optional.
- Deterministic fallback remains the supported path when Ollama is disabled or unavailable.
- Runtime inspector remains out of the default workflow.
- No autonomous self-improvement, web UI, or production harness mutation is implemented.
