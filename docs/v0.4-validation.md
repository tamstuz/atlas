# v0.4 Validation

v0.4 adds read-only runtime inspection and controlled execution planning.

## Endpoint

```bash
curl -s -X POST http://localhost:8088/projects/<PROJECT_ID>/runtime-inspect \
  -H "Content-Type: application/json" \
  -d '{"target_type":"unknown","target_hint":"","allow_read_only_commands":false}'
```

Expected response fields:

```json
{
  "project_id": "",
  "status": "complete",
  "runtime_inspection_report_path": "",
  "task_packet_path": "",
  "agent_result_path": "",
  "inspection_summary": "",
  "blockers": [],
  "evidence": [],
  "safe_to_modify": false
}
```

## Expected Artifacts

All runtime inspection artifacts must be project-local:

```text
/srv/ai-lab/projects/<project-id>/handoffs/runtime-inspector-task-packet.yaml
/srv/ai-lab/projects/<project-id>/handoffs/runtime-inspector-agent-result.json
/srv/ai-lab/projects/<project-id>/qa/runtime-inspection-report.md
/srv/ai-lab/projects/<project-id>/qa/runtime-inspection-evidence.json
/srv/ai-lab/projects/<project-id>/qa/candidate-runtime-registry-updates.yaml
```

v0.4 must not mutate global runtime registries.

## Command Execution

Default:

```env
RUNTIME_INSPECTION_COMMANDS_ENABLED=false
```

Command execution happens only when both are true:

- `.env` sets `RUNTIME_INSPECTION_COMMANDS_ENABLED=true`
- request JSON sets `allow_read_only_commands=true`

Only allowlisted read-only commands may run. Rejected and skipped commands must be recorded as evidence.

## Database Checks

Runtime inspector task:

```sql
select assigned_role, status
from tasks
where project_id = '<PROJECT_ID>' and assigned_role = 'runtime_inspector';
```

Runtime inspector agent run:

```sql
select
  ar.agent_role,
  ar.status,
  ar.output->>'safe_to_modify' as safe_to_modify,
  ar.output->'blockers' as blockers,
  ar.output->>'evidence_count' as evidence_count,
  ar.output->'evidence_summary' as evidence_summary,
  ar.output->>'fallback_used' as fallback_used
from agent_runs ar
join tasks t on t.id = ar.task_id
where t.project_id = '<PROJECT_ID>'
  and ar.agent_role = 'runtime_inspector'
order by ar.created_at;
```

## Known Limits

v0.4 is inspection and planning only. It does not implement live modification, approval-based execution, sudo-capable agents, self-improvement, or a web UI.
